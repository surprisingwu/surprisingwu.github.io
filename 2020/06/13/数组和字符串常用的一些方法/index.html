js继承的种类

JS 继承

许多OO语言中都支持俩种继承方式: 接口继承和实现继承.接口继承只继承方法签名,而实现继承则继承实际的方法. 由于JS中的函数没有签名,因此在ECMAScript中无法实现接口继承. ECMAScript只支持实现继承,而且依靠原型链来实现

原型链继承

核心思想: 子类的原型指向父类的实例

  • 缺点:
    • 所有的实例共享原型中的引用类型
    • 不能像父类的构造函数传参

借用构造函数

核心思路: 借用父类的构造函数

优点: 解决了共享引用类型的问题

  • 缺点
    • 共享的方法都要写在父类的构造函数中

组合继承

借用构造函数和原型链继承的组合

  • 优点
    • 解决了共享引用类型和传参的问题
  • 缺点
    • 调用俩次父类
    • 原型上属性重复

原型继承

借助原型, 根据已有的对象创建对象

  • 使用ES5提供的Object.create(obj, descriptorObj)也可以实现该效果
  • 优点
    • 根据已有的对象创建对象
      缺点:
      • 共享引用类型
      • 不能传参

寄生式继承

封装继承的过程, 并以某种方式增强对象

  • 能够复用执行对象的方法和属性, 并对其增强
  • 缺点
    • 没法传参
    • 增强的方法或属性没法复用
    • 共享引用类型

寄生组合继承

原型继承和寄生式继承的组合

类继承

es6新增类继承

  • 缺点:
    • 不能共享属性
    • 属性和方法重名时, 属性会屏蔽方法
